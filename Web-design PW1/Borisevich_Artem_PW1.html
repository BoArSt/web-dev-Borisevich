1. When HTML5 has become a standard? What were the main reasons for developing new standard?
HTML5 became a standard in October 2014. The main reasons for developing this new standard were to improve the functionality and interoperability of web pages, to support multimedia content without the need for plugins, and to provide better support for mobile devices and accessibility features.
2. What is a difference between W3C HTML5 and WHATWG HTML Living Standard?
The W3C HTML5 specification is a stable version that was finalized in October 2014, while the WHATWG HTML Living Standard is a continuously updated document that reflects ongoing changes and additions to the HTML language. The WHATWG version is more dynamic and flexible, allowing for rapid updates and improvements based on community feedback and evolving web technologies. Both versions aim to define the same set of standards for HTML, but the WHATWG version may include newer features and updates not yet incorporated into the W3C specification.

4. Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?
Semantic markup focuses on using HTML elements to convey the meaning and structure of content, making it more accessible to both users and search engines. Presentational markup, on the other hand, involves using HTML elements for styling and visual presentation, which can lead to less maintainable code and hinder accessibility.
Presentational markup is considered deprecated now because it mixes content with styling, making it harder to update and maintain. Separating content and presentation through CSS allows for more flexibility in design changes without affecting the underlying structure of the content. Semantic markup is preferred as it enhances accessibility, search engine optimization, and overall code quality.
5. How you can check browser support for particular tag or attribute?
You can check browser support for a particular HTML tag or attribute by using online resources like caniuse.com or MDN web docs. These websites provide detailed information on browser compatibility for various HTML elements, CSS properties, and JavaScript features. Additionally, you can use tools like BrowserStack or cross-browser testing services to test your website across different browsers and versions to ensure compatibility.
6. What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?
The HTML standard officially supports the following audio and video formats:
Audio:
- MP3
- WAV
- OGG
Video:
- MP4
- WebM
- OGG
If you use other formats that are not officially supported by the HTML standard, the audio or video may not play correctly or at all in some browsers. Browsers may not be able to decode or render unsupported formats, resulting in a blank space or an error message instead of the intended media content. To ensure cross-browser compatibility and proper playback, it is recommended to use the officially supported audio and video formats listed above when adding multimedia content to your website.
